之前一直以为，在js的位运算中，双精度浮点数在内部会先被转成32位整数，再执行位运算，然后再转为64位数值，因此效率极低。今天实验发现，对浮点执行位运算只会导致数值不准确，而不会导致效率低下，相比乘法，左移位要快66%（其实就是浮点运算与整数运算的差距）。
